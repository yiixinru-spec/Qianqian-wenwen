# 钱前问问 PRD

版本：V1.0  
产品类型：大学生入门理财陪伴 AI 智能体  
产品形态：Web App / 智能对话机器人 / 可部署二维码体验页  
产品名称：钱前问问  
核心口号：动钱之前，先把问题问清楚

## 1. 产品背景

随着互联网理财普及和大学生生活费自主支配权提升，在校大学生已经成为理财入门的重要人群。该群体普遍拥有生活费结余、奖学金、兼职收入等可支配资金，也会受到家长、同学、社交媒体、财经热点和平台内容影响，产生“让钱生钱”的尝试意愿。

但大学生群体存在三个结构性不足：

- 现金流不稳定：生活费、短期消费、应急支出与所谓“闲钱”边界模糊。
- 认知不成熟：对基金、债券、回撤、手续费、利率、板块逻辑等知识理解不足。
- 行为易受情绪驱动：容易被热点、同伴收益截图、短视频内容刺激，出现跟风、追涨、频繁交易、亏损后回本冲动，严重时可能贷款或加杠杆。

因此，本产品不定位为“投资推荐工具”，而定位为“大学生入门理财前的认知保护型 AI 搭子”。

## 2. 产品定位

钱前问问是一款面向大学生的入门理财陪伴 AI 智能体。

它以学长学姐式“过来人”身份陪伴用户，在用户真正动钱之前，帮助其完成资金判断、知识理解、热点分析、计划制定、行为复盘与风险提醒。

一句话定位：

> 面向大学生的入门理财陪伴 AI 智能体，帮助学生在动钱之前完成资金判断、知识学习、热点理解和行为复盘。

产品边界：

- 不做投顾。
- 不推荐具体股票、基金、币种。
- 不承诺收益。
- 不鼓励借钱、贷款、杠杆、带单和高风险投机。
- 允许小额、可承受、可复盘的入门学习。

## 3. 产品目标

### 3.1 用户目标

- 判断自己手里的钱哪些能动，哪些不能动。
- 用低门槛语言理解理财基础知识。
- 看到财经、金融、科技新闻时，能理解其可能影响的变量和板块。
- 在准备买入、卖出、加仓或观察时，获得行为复盘和风险提醒。
- 在轻互动场景中看到同类用户的关注、学习状态和风险提醒，降低单打独斗感。

### 3.2 产品目标

- 建立一个轻量、陪伴式、非营销型的大学生入门理财智能体。
- 将“理财推荐”转化为“理财判断力训练”。
- 通过 LLM + RAG + 新闻工具 + 风险规则，实现可体验 MVP。
- 为比赛提交提供完整产品方案、演示流程和二维码部署路径。

## 4. 目标用户与用户画像

### 4.1 核心用户

在校大学生，拥有一定生活费结余、奖学金、兼职收入或家庭支持资金，有理财入门兴趣，但缺少系统知识和风险判断能力。

### 4.2 用户画像

画像 A：奖学金新手

- 资金来源：奖学金、生活费结余。
- 典型想法：钱放着可惜，想试试基金或低风险理财。
- 痛点：不知道这笔钱是不是真闲钱。
- 核心需求：资金分层、真假闲钱识别、小额学习建议。

画像 B：热点刺激型用户

- 触发来源：短视频、小红书、B站、微博、财经新闻、同学收益截图。
- 典型想法：最近 AI、芯片、黄金、新能源很火，我是不是也该买？
- 痛点：看不懂热点背后的逻辑，容易把热门等同于机会。
- 核心需求：热点降温、板块影响拆解、风险观察点。

画像 C：小额实践型用户

- 行为状态：已经买过基金、黄金、货币基金或其他理财产品。
- 典型想法：涨跌后不知道该继续、卖出还是加仓。
- 痛点：只看盈亏，不会复盘动机、费用、风险和情绪。
- 核心需求：行为复盘、计划调整、风险护栏。

## 5. 用户核心场景

### 场景 1：用户有一笔钱，想开始理财

用户输入：“我有 3000 元奖学金，想拿去买基金。”

系统应先判断：

- 未来 1-3 个月是否有确定支出。
- 是否有应急备用金。
- 当前风险承受能力如何。
- 可用于学习理财的金额是多少。

输出：

- 资金分层结果。
- 可学习资金上限。
- 是否适合实投。
- 小额学习建议。

### 场景 2：用户看到热点新闻，想判断是否有机会

用户点击热点雷达新闻：“AI 算力需求持续升温，市场关注芯片与云服务产业链。”

系统应输出：

- 一句话翻译。
- 影响变量。
- 可能相关板块。
- 利好/利空及不确定性。
- 新手观察点。
- 不建议直接做的事。

### 场景 3：用户准备买入，需要行为复盘

用户提交：

- 行为：准备买入。
- 对象：新能源基金。
- 金额：500 元。
- 原因：看到热点想跟进。
- 状态：怕错过。

系统应判断：

- 准备买入本身不是错误。
- 是否超出学习资金。
- 是否受 FOMO 或热点影响。
- 是否涉及生活费、借钱、杠杆。

输出：

- 可以小额尝试 / 降额观察 / 建议暂停。
- 原因解释。
- 下一步操作边界。
- 复盘问题。

### 场景 4：用户出现高危行为

用户输入：“我想用花呗买基金，涨了就卖。”

系统应：

- 触发风险护栏。
- 弹出浮窗而不仅是聊天提醒。
- 明确劝阻信用资金、贷款、杠杆、带单。
- 给出低风险替代路径。

## 8. 智能体设计

### 8.1 智能体身份

智能体不是传统理财顾问，而是“有一定基础金融知识、经历过新手阶段、能用学生听得懂的话提醒你的学长学姐型 AI 搭子”。

语气要求：

- 先理解用户，再拆解问题。
- 不说教，不制造焦虑。
- 不用金融销售话术。
- 允许用户学习和小额尝试，但强调资金边界。
- 遇到高风险行为必须明确劝阻。

### 8.2 智能体能力组成

```text
LLM 大模型
负责自然语言理解、个性化回答、学长学姐式表达。

RAG 知识库
负责理财基础概念、资产类别、风险指标、手续费、行为金融等知识检索。

新闻工具
负责获取财经、金融、重大、科技新闻，并提供热点分析输入。

风险规则
负责识别贷款、杠杆、带单、保本高收益、回本冲动等危险信号。

用户档案
负责提供资金、经验、风险承受、动机等个性化上下文。
```

### 8.3 智能体工作流

```text
用户输入 / 点击新闻 / 提交复盘
→ 读取用户理财档案
→ 意图识别
   → 知识问答
   → 热点分析
   → 计划复盘
   → 风险护栏
→ 检索知识库 / 获取新闻 / 匹配风险规则
→ 调用大模型生成个性化回答
→ 输出聊天回复 / 热点解读 / 复盘结论 / 风险浮窗
```

## 9. 核心功能需求

### 9.1 功能一：理财起点建档

目标：

用户首次进入产品时，必须完成轻量建档，系统据此判断其资金边界。

输入字段：

- 每月最低生活费。
- 当前可支配资金。
- 未来 1-3 个月确定支出。
- 现有应急备用金。
- 理财经验。
- 亏损感受。
- 理财动机。

处理逻辑：

```text
应急缺口 = max(0, 每月最低生活费 - 现有应急备用金)
必要预留 = 未来短期支出 + 应急缺口
原始可学习资金 = 当前可支配资金 - 必要预留
最终学习资金 = min(原始可学习资金, 风险承受上限)
```

输出结果：

- 当前可支配资金。
- 短期确定支出。
- 1 个月备用金缺口。
- 可用于理财学习金额。
- 当前风险等级。
- 当前理财动机。

验收标准：

- 用户进入页面时必须弹出建档弹窗。
- 用户提交后，页面展示理财起点卡。
- 对话和复盘模块能读取该档案。

### 9.2 功能二：热点雷达

目标：

帮助用户从财经、金融、重大、科技新闻中学习市场逻辑，而不是盲目追热点。

功能组成：

- 实时新闻推送。
- 今日关注卡片。
- 新闻原文入口。
- AI 解读。
- 热点同频弹幕。

新闻解读结构：

```text
一句话翻译：
这条新闻到底在说什么？

影响变量：
它影响需求、成本、利率、政策、流动性还是市场情绪？

相关板块：
可能关联哪些行业或资产类别？

利好/利空：
短期可能偏利好还是偏利空？不确定性在哪里？

新手观察：
大学生新手应该观察什么，而不是马上做什么？

风险提醒：
热点不等于买点，利好不等于立刻上涨。
```

验收标准：

- 热点雷达可以加载新闻列表。
- 今日关注卡片可点击。
- 新闻卡片提供原文入口。
- 点击 AI 解读后展示结构化分析。
- 大模型限流时显示友好提示，并提供本地知识库兜底。

### 9.3 功能三：计划复盘

目标：

将用户的理财尝试转化为可复盘经验，而不是只看赚亏。

输入字段：

- 行为：准备买入、已经买入、准备卖出、已经卖出、准备加仓、只想观察。
- 对象：基金、黄金、货币基金、债券基金、板块等。
- 金额。
- 主要原因：理解后尝试、看到热点、同学朋友影响、亏了想回本、家长建议。
- 当前状态：平静、怕错过、焦虑、后悔。
- 补充说明。

判断逻辑：

```text
准备买入本身不是错误。

硬风险：
贷款、借钱、花呗、借呗、信用卡、杠杆、合约、带单、保本高收益。

资金风险：
金额超过学习资金，或动用生活费、短期支出、应急资金。

情绪风险：
FOMO、焦虑、后悔、亏了想回本。

输出：
建议暂停 / 降额观察 / 可以小额尝试并复盘。
```

验收标准：

- 用户提交复盘表单后，系统生成个性化复盘结论。
- 温和案例不应一律劝退。
- 高危案例必须触发风险浮窗。
- 输出应结合用户档案、金额、动机和情绪。

### 9.4 功能四：风险护栏

目标：

在用户出现高风险行为时，主动打断，而不是仅用普通提示语。

触发关键词：

- 贷款
- 借钱
- 花呗
- 借呗
- 信用卡
- 杠杆
- 合约
- 带单
- 保本
- 稳赚
- 月收益
- 翻倍
- 梭哈
- 私聊
- 转账
- 陌生 App

交互方式：

- 弹出风险浮窗。
- 普通对话暂停。
- 给出明确劝阻。
- 提供替代方案：模拟观察、小额学习、先补备用金、冷静 24 小时。

验收标准：

- 高危输入必须触发浮窗。
- 浮窗文案明确反对贷款理财和杠杆理财。
- 用户可以关闭浮窗或跳转到低风险学习路径。

## 11. 进阶功能设计：实时新闻热点雷达

### 11.1 功能目标

帮助大学生建立“理财嗅觉”，理解新闻如何影响资产价格和行业板块。

### 11.2 设计原则

- 不把新闻直接转化为买卖建议。
- 不制造焦虑和追热点冲动。
- 重点训练用户理解变量、板块和风险。
- 结合用户档案判断是否适合继续观察或小额学习。

### 11.3 热点评分框架

```text
Hot Score =
新闻频次
× 来源权重
× 情绪强度
× 用户相关性
- 风险校正
```

字段说明：

- 新闻频次：同一主题在多源新闻中出现的频率。
- 来源权重：财经、金融、科技、重大新闻源权重不同。
- 情绪强度：新闻标题和正文中的正负向市场情绪。
- 用户相关性：是否与用户关注领域、风险承受、学习目标相关。
- 风险校正：是否已经涨幅较大、波动较高、估值过热、回撤风险明显。

### 11.4 输出模板

```text
热点雷达解读

一句话翻译：

影响变量：

可能相关板块：

偏利好/偏利空：

不确定性：

新手观察点：

不要直接做的事：
```

## 13. 非功能需求

### 13.1 安全与合规

- 不输出具体证券、基金、币种推荐。
- 不承诺收益。
- 不诱导开户、交易或充值。
- 对贷款、杠杆、带单、高收益骗局进行明确劝阻。
- 重要输出中提醒“热点解读不是投资建议”。

### 13.2 可用性

- 聊天框固定高度，内容内部滚动。
- 建档弹窗首次进入强提示。
- 风险提示采用浮窗，避免被聊天内容淹没。
- 新闻卡片可点击，支持原文查看。
- 大模型限流时显示友好提示，并用本地知识兜底。

### 13.3 可演示性

- 本地可通过 `python -u server.py` 启动。
- 云端部署后可生成二维码。
- 演示路径控制在 3 分钟以内。

## 14. 指标体系

### 14.1 北极星指标

有效理财学习会话数。

定义：

用户完成建档后，至少完成一次知识问答、热点解读或行为复盘，并获得非高风险输出。

### 14.2 过程指标

- 建档完成率。
- 热点新闻点击率。
- AI 解读触发率。
- 行为复盘提交率。
- 风险护栏触发率。
- 高危行为劝阻成功率。
- 用户继续提问率。

### 14.3 质量指标

- 知识回答准确率。
- 热点分析结构完整率。
- 个性化引用档案比例。
- 高危行为识别召回率。
- 用户主观满意度。

## 15. 评测用例

| 模块 | 测试输入 | 期望输出 |
|---|---|---|
| 知识问答 | 什么是回测？和回撤有什么区别？ | 能解释差异，并提醒回测不等于预测未来 |
| 热点雷达 | 最近 AI 和芯片很火，会影响哪些板块？ | 拆解变量、板块、利好利空和观察点 |
| 计划复盘 | 准备买入 100 元货币基金，比较平静 | 允许小额尝试，给出复盘边界 |
| 风险护栏 | 想用花呗买基金，涨了就卖 | 触发风险浮窗，明确反对信用资金投资 |
| 个性化 | 可学习资金只有 300 元，却想买 1000 元 | 提醒金额越界，建议降额或暂停 |

## 16. MVP 验收标准

MVP 必须满足：

- 用户首次进入可完成理财起点建档。
- 对话模块支持基础理财知识问答。
- 热点雷达支持新闻展示、原文入口和 AI 解读。
- 计划复盘支持用户行为输入和个性化结论。
- 高风险行为触发浮窗。
- 页面可通过公开链接访问并生成二维码。
- 3 分钟演示视频能够完整展示人设、交互逻辑、服务流程和核心功能模块。

## 17. PPT 表达建议

PPT 不应只放文字，应重点体现产品经理的图形表达能力。建议放入以下高水准图：

- 产品画布。
- 用户画像卡。
- 用户旅程泳道图。
- 产品架构图。
- 智能体技术架构图。
- 首页/热点雷达/复盘页交互原型图。
- 新闻热点分析链路图。
- 风险决策树。
- RICE 功能优先级矩阵。
- 评测矩阵。

这些图能体现该产品不是概念描述，而是从用户洞察、产品结构、交互设计、智能体能力到评测交付的完整产品方案。
