# 可体验二维码发布方案

本地 `python server.py` 只适合开发和调试，不能作为最终提交方式。赛题要求“可体验二维码”，意味着需要一个公网可访问链接，再把链接生成二维码。

## 推荐方案一：智能体平台发布

最适合比赛提交。

使用平台：

- 扣子
- Dify
- 通义智能体
- 智谱智能体

操作：

1. 创建智能体，名称填写“钱前问问”。
2. 复制 `agent/system_prompt.md` 到系统提示词。
3. 上传 `knowledge/` 下的知识库文档。
4. 配置快捷入口：理财起点、热点雷达、小额计划、风险护栏。
5. 如果平台支持联网工具，接入新闻搜索；不支持则保留“用户粘贴新闻分析”。
6. 发布智能体，复制公开体验链接。
7. 用公开体验链接生成二维码，放进 PPT。

优点：

- 不需要用户打开终端。
- 天然支持二维码。
- 更符合“智能体搭建”赛题要求。

## 推荐方案二：部署本项目 Web MVP

如果想展示当前网页原型，需要把 `campus-finance-buddy` 部署到云端。

可选平台：

- Render
- Railway
- 腾讯云 CloudBase
- 阿里云函数计算
- 自有服务器

启动命令：

```bash
python server.py
```

环境变量：

```text
PORT=平台自动提供
HOST=0.0.0.0
LLM_API_KEY=你的大模型 Key，可选
LLM_BASE_URL=OpenAI-compatible Chat Completions 地址，可选
LLM_MODEL=模型名称，可选
```

部署成功后，会得到一个公网 URL，例如：

```text
https://your-app.example.com
```

把这个 URL 生成二维码即可。

## 二维码放在 PPT 哪里

建议放在第 11 页“智能体搭建与评测”或第 12 页“总结与体验入口”。

展示格式：

```text
扫码体验钱前问问
面向大学生的入门理财陪伴智能体
```

## 注意

不要提交 `http://127.0.0.1:8790/` 的二维码。这个地址只能在你自己的电脑上访问，评委扫码打不开。

## 本项目已准备好的部署文件

```text
Dockerfile
Procfile
render.yaml
requirements.txt
```

这些文件用于云平台识别启动方式。部署后访问公网 URL 即可体验页面。
